Quick PNG explanation

A PNG is composed of many Chunck which contains informations about the image.
A header (not a Chunck), IDHR which contains informations such has width, height and bit depth, PLTE which is optionnal depending on the type of image but it is needed for my program to work.
Then we have the IDAT chunck with the image data and IEND chunck which marks the end of the image.

Header -> IDHR -> (PLTE) -> IDAT* -> IEND

(Those are only the critical chunck which cannot be ignored when parsing)

On a color type 3 image (indexed color) colors are stored in a Palette in the form (R,G,B) which is then associated to each pixel in the IDAT chunk. Meaning the first color to appear in the Palette is gonna have index 0 and the pixel needing this color will be associated with it

What my program does

Briefly, my program parses the PNG image, checks if it is indeed a PNG and if it is of color type 3 and then displays all colors present in the PLTE Chunck (so all the color present in the image), it then asks user which one to remplace and with what color.
I then replace the bytes in the image and update the CRC (which is a way to check no data was lost).
As I said in the Quick explanation each pixel in the image is associated with its color from the Palette, meaning that changing one color
changes every pixel associated with it, as you can see in the example images the mouth as also changed color because I changed all red pixel to purle
